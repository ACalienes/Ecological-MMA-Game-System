# Design Rules

<div class="emma-meta"><span>Theory Layer</span><span>20 Working Rules</span><span>What Every Game Obeys</span></div>

The practical rule set — the takeaways from [the principles](principles.md) and [the evidence](landmark-studies.md) that every game, constraint, and content decision must obey. Grouped into four families. Each rule carries its printed-page citation to [Blau &amp; Wagman (2022)](index.md).

---

<p class="emma-glabel">Constraints & Scaling</p>

<p class="emma-hl">Build the problem from the environment. Constrain the option space so functional behavior emerges, and scale the problem to the body — not to absolute numbers.</p>

<ul class="emma-checklist">
  <li><b>1. The unit is the fighter–opponent–cage system</b> — never the fighter alone. Every game includes an adaptive opponent and live environmental constraints (boundary, range). <i>(p. 57)</i></li>
  <li><b>2. Constrain to afford; never prescribe.</b> A constraint reduces the option space so behavior emerges — "outlets every 12 feet," not "outlet here." If a rule forces one solution, it is over-constraining. <i>(p. 146, 271)</i></li>
  <li><b>3. Scale constraints by critical ratios, not absolutes.</b> Distances, heights, apertures, and ranges are set as ratios of the athlete's reach / leg length / shoulder width, so different bodies face the same relational problem. <i>(Warren, p. 79–86)</i></li>
  <li><b>18. Train with the gear, in motion, with consequence.</b> Affordances are for the animal-plus-object system and change with gloves, fatigue, and weapons; a moving fighter occupies more space; emotion is nested in the system. <i>(p. 57, 86–87)</i></li>
  <li><b>19. Design affordances and information into the environment; prefer information over instruction.</b> Make desired behavior perceptually obvious (live boundary, clear range markers); manipulate information to compel behavior (Chicago's optic-flow lines → 36% fewer crashes). <i>(p. 211–216)</i></li>
</ul>

<p class="emma-glabel">Information & Perception</p>

<p class="emma-hl">Name what the athlete attunes to, and reward the opportunity actually taken — not isolated cues, and never an internal computation.</p>

<ul class="emma-checklist">
  <li><b>4. Name the information the athlete must attune to, per game.</b> Specify the invariant — τ (time-to-contact) for closing strikes/shots, global optic flow for self-motion/cage approach, the inertial array (resistance-to-rotation) for grips/clinch. "Reading" = attuning to a named invariant. <i>(p. 82, 123–127, 165–166)</i></li>
  <li><b>5. Reward the taken opportunity (affordance), not isolated cues.</b> Score the realized behavior — a clean counter, a completed off-balance — because an affordance is a simultaneous multi-property fit perceived in itself, not a single property. <i>(p. 77, 87–88)</i></li>
  <li><b>6. Reward economical / preferred solutions, and let athletes find their own transition points.</b> Perception is action-scaled to effort; skilled actors transition at a comfortable preferred boundary before their max. Phase-advance triggers are the athlete's perceived readiness, not a coach's command. <i>(p. 81, 88)</i></li>
  <li><b>7. Never decompose the perception–action loop.</b> Simplify by scaling the whole task (slower, smaller, lighter, fewer options); never break perception off from action. Static unopposed drilling removes the information the skill needs. <i>(p. 192, 207)</i></li>
  <li><b>15. Design opponent-reading explicitly: perceive affordances for self AND opponent.</b> Reward recognizing what the opponent's position affords <i>them.</i> <i>(p. 175)</i></li>
  <li><b>16. Build distance games on two preferred ranges + a dead middle.</b> Near affords offense, far affords defense, the middle affords neither — reward living in the advantageous range; experienced fighters spend more time in the offensive range. <i>(kendo, p. 188–189)</i></li>
</ul>

<p class="emma-glabel">Learning & Transfer</p>

<p class="emma-hl">Let solutions self-assemble, repeat the problem rather than the movement, treat skill as an attractor layout — and never assume a skill carries across action modes.</p>

<ul class="emma-checklist">
  <li><b>8. Let synergies soft-assemble to the goal; cue cooperation with free forces.</b> "Points for staying standing," not "you must sprawl." Cue base, suspension, redirecting directional force — capitalize on gravity and reactive forces ("controlled falling"). <i>(p. 143–146)</i></li>
  <li><b>9. Repeat the problem, not the movement.</b> Coordination stage: repeat the same problem to assemble the synergy. Mastery stage: repeat the goal across varied opponents/contexts. Variability is the feature. <i>(p. 145, 169–170)</i></li>
  <li><b>10. Treat skill as an attractor layout the coach reshapes — and distinguish learning from adaptation.</b> Learning = birth of a new attractor (exploration, expect instability). Adaptation = moving an existing attractor to a new context. Name your control parameters (speed, pressure, range, fatigue, # opponents) and order parameter (what you measure). <i>(p. 234–251)</i></li>
  <li><b>11. Use noise / instability on purpose.</b> The "amber zone" is functional noise that lets a fighter escape a shallow habit and find a deeper pattern. Perturb working patterns and reward re-organization. <i>(p. 242–243, 272–273)</i></li>
  <li><b>12. Expect nonlinear switches under a control parameter — design for them.</b> Stance, rhythm, and gait reorganize abruptly past a critical value; minor rule changes revolutionize styles because they cross a threshold. <i>(p. 152–155, 250)</i></li>
  <li><b>13. Build learning paths as: educate intention → educate attention → calibrate.</b> Choose the affordance, attune to the invariant that specifies it, then scale the response with feedback (reduce feedback over time). <i>(p. 204–206)</i></li>
  <li><b>14. Assume NO transfer across action modes; verify transfer empirically.</b> Reading τ for striking ≠ reading τ for takedowns; build separate per-mode paths and test. Phase-scaffold (striking → clinch → takedown) precisely because it re-educates attention at each new mode. <i>(p. 160–161, 206)</i></li>
</ul>

<p class="emma-glabel">Coaching Stance</p>

<p class="emma-hl">The coach is an environment architect and rule-giver — not a choreographer. Keep the vocabulary disciplined and grounded.</p>

<ul class="emma-checklist">
  <li><b>17. Coach by giving local rules and letting athletes adapt — not a blueprint.</b> "Give the contractors a list of rules… the houses will be far more adapted to their circumstances, and you will have to do a lot less work." <i>(p. 284–285)</i></li>
  <li><b>20. Keep the vocabulary disciplined and grounded.</b> Maintain controlled vocabulary; add a sixth narration input to game capture — <i>the information the athlete must attune to.</i> Reserve "adaptation" for moved-attractor work and "learning" for new-attractor work. <i>(p. 241)</i></li>
</ul>
