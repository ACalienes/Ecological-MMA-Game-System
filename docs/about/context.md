# Project Context & Goals

<div class="emma-meta"><span>About</span><span>The Project Story</span><span>Intent · Philosophy · Structure</span></div>

<p class="emma-hl"><b>The big idea:</b> we are building a <b>constraints-led, ecological training framework for MMA</b>, not a catalogue of moves, but a library of <i>repeating fight problems</i> and the environments that shape the solutions.</p>

<p class="emma-glabel">What We Are Building</p>

<div class="emma-cards">
  <div class="emma-card"><div class="emma-card__i">📚</div><div class="emma-card__t">A knowledge base</div><p>Structured games and concepts, authored in Markdown.</p></div>
  <div class="emma-card"><div class="emma-card__i">🗺️</div><div class="emma-card__t">A navigable site</div><p>Rendered as a documentation site you can move through.</p></div>
  <div class="emma-card"><div class="emma-card__i">🥋</div><div class="emma-card__t">Built for the mat</div><p>Organized for practical use in real training environments.</p></div>
</div>

<p class="emma-hl"><b>Key distinction:</b> the system is <b>game-based</b>, not technique-based. We are not cataloging moves, we are organizing <i>repeating MMA problems</i> and the environments that shape them.</p>

<p class="emma-glabel">Core Training Philosophy</p>

<div class="emma-cards">
  <div class="emma-card emma-has-pop" tabindex="0"><div class="emma-card__i">🧠</div><div class="emma-card__t">MMA is a decision system</div><p>The system is built around recurring <b>decision states</b>, not named techniques.<span class="emma-pop">Fights repeat predictable patterns: accessing an opponent, stabilizing position, exploiting with strikes or submissions, and countering escapes or reversals.</span></p></div>
  <div class="emma-card emma-has-pop" tabindex="0"><div class="emma-card__i">🌿</div><div class="emma-card__t">The environment teaches</div><p>Using <b>Representative Design</b>, behavior emerges from constraints, the coach designs the environment, not the solution.<span class="emma-pop">Athletes learn by solving problems, not following instructions. Games are constructed so correct behavior is rewarded naturally.</span></p></div>
  <div class="emma-card emma-has-pop" tabindex="0"><div class="emma-card__i">📍</div><div class="emma-card__t">Control precedes damage</div><p>Across all environments: <b>control before striking</b>, <b>stability before exploitation</b>.<span class="emma-pop">Strikes are allowed in most games, but they are informational and carry positional trade-offs.</span></p></div>
  <div class="emma-card emma-has-pop" tabindex="0"><div class="emma-card__i">⚖️</div><div class="emma-card__t">Every offense needs a defense</div><p>The system is built symmetrically, no domination layer exists without resistance.<span class="emma-pop">If we teach pinning, we teach escaping. If we teach takedowns, we teach scrambles.</span></p></div>
</div>

<p class="emma-glabel">Design Rule: Games vs Levels</p>

<div class="emma-flow">
  <div class="emma-flow__row"><span class="emma-flow__from">Environment or dominant constraint changes</span><span class="emma-flow__arr">→</span><span class="emma-flow__to">New <b>game</b></span></div>
  <div class="emma-flow__row"><span class="emma-flow__from">Difficulty or completion increases</span><span class="emma-flow__arr">→</span><span class="emma-flow__to">New <b>level</b></span></div>
</div>

<div class="emma-chips">
  <span class="emma-chip emma-chip--info" tabindex="0"><span class="emma-chip__i">🎮</span> Game<span class="emma-chip__detail">A distinct learning problem with a stable environment, clear goal, defined invariants, and clear win conditions.</span></span>
  <span class="emma-chip emma-chip--info" tabindex="0"><span class="emma-chip__i">📈</span> Level<span class="emma-chip__detail">A progression inside the same game with stricter success criteria, same problem, higher standard.</span></span>
</div>

<p style="font-size:.82rem;color:var(--fog-dim);margin:.7rem 0 0;">Full breakdown → <a href="../levels-vs-games/">Levels vs Games</a></p>

<p class="emma-glabel">End State Vision</p>

<ul class="emma-checklist">
  <li><b>Comprehensive</b> ecological MMA curriculum</li>
  <li><b>Navigable</b> via visual map</li>
  <li><b>Usable</b> by coaches as a reference and planning tool</li>
  <li style="opacity:.6;">Complete coverage of all MMA environments and decision states <i>(in progress)</i></li>
</ul>

<p class="emma-glabel">System Evolution Philosophy</p>

<p class="emma-hl">This framework is intentionally <b>non-dogmatic</b>. As new information, competitive insight, or safety considerations arise, concepts may be refined, definitions updated, and games reorganized.</p>

<p class="emma-sub">The only non-negotiables</p>

<div class="emma-cards">
  <div class="emma-card"><div class="emma-card__i">🌍</div><div class="emma-card__t">Ecological validity</div><p>It must hold up in real fighting contexts.</p></div>
  <div class="emma-card"><div class="emma-card__i">🧩</div><div class="emma-card__t">Internal consistency</div><p>The pieces must agree with one another.</p></div>
  <div class="emma-card"><div class="emma-card__i">🦺</div><div class="emma-card__t">Safety</div><p>No game compromises participant safety.</p></div>
  <div class="emma-card"><div class="emma-card__i">🎯</div><div class="emma-card__t">Clarity of objectives</div><p>Every game states what it develops.</p></div>
</div>
