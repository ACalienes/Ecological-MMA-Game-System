## The big finding: we have THREE games in the same family, all WIP

`the-square`, `one-shot-discovery`, and the un-built 2026-05-31 G2–G6 are **siblings of the same striking-in-a-tight-perimeter game**, evolved across three classes. They share:

- tight rectangular perimeter, both feet out = loss
- role switch on success
- progressive constraint layering ("earn the right to counter" then "earn the right to finish")
- counter must LAND (not just be thrown)
- feints as counter openings

What's different between them is **one or two knobs**:

| Game | Offense | Defense scoring | Counter constraint | Domain reach | Page status |
|---|---|---|---|---|---|
| **the-square** | ≤3-shot boxing combo | block-only L1 → counter L2 → clinch L3 → kick L4 → takedown L5 | one strike → body kick | striking → clinch → **ground (takedown finish)** | WIP, exists |
| **one-shot-discovery** | single shot, 4 weapons, no-repeat | block-only L1 → vary L2 → clinch as defense L3 → counter L4 → body-kick counter L5 | any single strike → body kick | striking only | WIP, built 2026-05-30 |
| **combo-reset** (unbuilt) | 4-shot combo, "every exchange is a result" | block-only L1 → counter L2 (1pt/5pt scoring) → opposite-side 2-strike L3 → kick-finished L4 → counter-only-wins + non-scoring jab L5 | any single → 2-strike opposite side → kick-finished → counter-only-wins | striking only | un-built, in mining report |

Reading these as iterations of the same idea, the family has been refining around four constraint dimensions:
- **offense complexity** (single shot / ≤3 / 4-shot combo)
- **counter constraint** (any single / opposite-side / kick-finished)
- **defense win paths** (block-only / block-or-counter / counter-only + deny tool)
- **domain extension** (striking only / + clinch / + takedown)

## Three possible structures (your call)

### Option A — One canonical page, the others fold in

Pick **`the-square`** as canonical (it has the broadest domain reach and the most ambitious "earn the right to finish" arc). Fold the best constraint additions from `one-shot-discovery` (single-shot variant) and 2026-05-31 (combo offense, point-weighted scoring, opposite-side counter, non-scoring jab) into it as Levels or variants.

- **Pro:** one page = one game in the library. Matches "core foundation that works best" most strictly. Curated, not cataloged.
- **Con:** the-square page would get long (5 → 8–10 levels), and a "single-shot vs combo" choice in the same game is awkward unless we add a parameter at the top.
- **Throwaway data:** the G1–G6 iteration record stays in the mining report. `one-shot-discovery` deprecated; its unique L2 "vary the tool" rule absorbed.

### Option B — Two canonical pages, one family

Keep **`the-square`** (the full domain-spanning progression) and build **one new combo-focused page** for today's G2–G6 (different enough — combo offense + non-scoring deny tool + point-weighted scoring). Deprecate `one-shot-discovery` (the single-shot version is closer to the-square's early levels than to today's combo version).

- **Pro:** two distinct game pages each with a clear unique selling point. The-square = full progression; combo-page = striking-focused with discrete-exchange + non-scoring deny tool. Less overlap.
- **Con:** still leaves the single-shot iteration as "data not shipped" — feels like a sunk cost. (But that's exactly the curated-product rule in action.)
- **Throwaway data:** one-shot-discovery deprecated, plus its hero image.

### Option C — Three canonical sister pages + a family page

Keep all three (the-square + one-shot-discovery + new combo-page) as siblings in a *Perimeter Exchange family*. Each occupies a different point in the matrix (single-shot vs combo vs full-domain-progression). A thin family page connects them with a comparison matrix; the user picks based on what they want to train that day.

- **Pro:** preserves the work, honors that each iteration has a real unique value (single-shot is genuinely different from combo; the-square's domain extension is genuinely different from both). Maps cleanly onto the cocktail-genealogy / comparison-matrix pattern we landed on yesterday.
- **Con:** three pages in one family is approaching "BJJ-Fanatics" anti-pattern unless the family page does real navigation work.

**My honest read:** Option B is the cleanest. The single-shot variant (`one-shot-discovery`) was a class-iteration step on the way to the combo version; the combo version is the more representative MMA reality. The-square is unique because of its domain extension and stands alone.
