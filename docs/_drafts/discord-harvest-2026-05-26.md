## Reusable design PATTERNS extracted (worth more than any single game)

1. **The control ladder** — same position, rungs that progressively increase the dominant side's control (frames → +underhook → +cross-face). Gives a built-in difficulty scale and clean per-rung win/loss. Adopt across ALL per-position games.
2. **The disc/dot objective** — "touch/step on a marker ~10 ft away in N seconds" turns a fuzzy escape into a binary, spectator-legible win. Solves our observable-win QA requirement elegantly.
3. **EMOM start-swaps inside one container** — one game, rotating typed starts each minute. A way to cover several clinch types under one game shell.
4. **Three-simultaneous-win-paths** — disconnected start, multiple legal wins (head / leg / body-lock). Reads affordances rather than scripting one answer.

## New portable meta-frameworks from Batch 2 (multiply into many games)

5. **Sub-control framework (breanne)** — start in any submission, forbid the original finish, win by *maintaining or upgrading* (to back or a different sub); loser escapes. One template → a control game for every sub class. Highest-leverage single idea in either batch.
6. **No-C2C control (breanne)** — control a pin via levers/wedges (hand-by-head + hand-by-hip) instead of chest pressure. A whole alternative control pedagogy for side/KOB/mount.
7. **Continuous-vs-terminal asymmetry** — one player has a terminal win, the other just maintains/survives for time (K-guard hold, KOB base, seatbelt cycle). Cleaner than a symmetric start; a default shape for "maintenance" games.
8. **Hidden-objective / secret-missions layer** — a deck of private win conditions over any open roll. This is a *product feature*, not just a game (gamified live rounds).
9. **Disc/marker objective (confirmed core)** — recurs across dot-stand, KOTH, landmines, salsa, touch-the-foot. The canonical way to make off-balancing/footwork wins observable.
